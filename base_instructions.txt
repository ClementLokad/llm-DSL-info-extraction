Envision est un DSL (Domain Specific Language) développé par Lokad pour l'analyse et l'optimisation prédictive sur des données relationnelles métier, typiquement pour des besoins de Supply Chain. Il intègre la programmation vectorielle (array programming). Il est fortement typé, impératif, fonctionnel, auto-distribué, avec une syntaxe rappelant Python et SQL.

Un script lit des fichiers plats (tables) en entrée depuis le système de fichiers Lokad et produit un tableau de bord web en sortie. Le tableau de bord est constitué de tuiles (tiles), la structure de base d'Envision pour composer un tableau de bord. La sortie peut également inclure des fichiers plats écrits sur le système de fichiers Lokad.

Structure et Logique :

    Style Python/SQL : Langage strict sur l'indentation (2 espaces pour entrer dans un bloc with, where, etc.). Opérations vectorielles (comme SQL) : Table.A + Table.B calcule toute la colonne d'un coup.

    Données (I/O) : read charge les fichiers (CSV, ION) dans des tables typées. show affiche les résultats sous forme de "tuiles" (graphiques, tableaux) dans le dashboard.

Variables et Tables :

    Vecteurs vs Scalaires : Une variable avec un point (Orders.Montant) est un vecteur (colonne). Sans point (tauxChange), c'est un scalaire (valeur unique).

    Dimensions : Les tables ont souvent une clé primaire entre crochets (ex: Items[id]). Les tables calendaires (Day, Week) sont générées automatiquement via expect [date].

Manipulations Clés :

    Filtrage : where restreint les données pour le bloc indenté suivant. keep where applique le filtre jusqu'à la fin du script.

    Agrégation : sum, count, max réduisent les données. La syntaxe sum(T.Val) by T.Group correspond au "GROUP BY".

    Broadcasting : Les données d'une table parente ("upstream") ou scalaires sont automatiquement propagées vers les tables enfants ("downstream") sans jointure explicite.

    Fonctions : def pure définit une fonction classique, def process définit un agrégateur personnalisé avec mémoire.
